pingu.corpus.io¶

The io module contains the loaders which provide functionality to load corpora from the filesystem.

All loader implementations base on pingu.corpus.io.CorpusLoader.

pingu.corpus.io.available_loaders()[source]¶

Get a mapping of all available loaders.

Returns:A dictionary with loader classes with the name of these loaders as key.
Return type:dict

Example:

>>> available_loaders()
{
    "default" : pingu.corpus.io.DefaultLoader,
    "kaldi" : pingu.corpus.io.KaldiLoader
}
pingu.corpus.io.create_loader_of_type(type_name)[source]¶

Create an instance of the loader with the given name.

Parameters:type_name – The name of a loader.
Returns:An instance of the loader with the given type.

Loader implementations¶

Default¶

class pingu.corpus.io.DefaultLoader[source]¶

This is the corpus loader which is used by default.

Broadcast¶

class pingu.corpus.io.BroadcastLoader[source]¶

This is the corpus loader which is used for corpora where a separate label file per utterance exists. This especially is useful for corpora where the utterances are very long (e.g. broadcast recordings).
